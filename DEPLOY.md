# Deploy Ghar Seva — Netlify (frontend) + Render (API + PostgreSQL)

## Architecture

| Layer | Platform | What runs |
|--------|----------|-----------|
| Frontend | **Netlify** | Static site (`frontend/`) |
| API | **Render** Web Service | Express + JWT auth (`backend/`) |
| Database | **Render** PostgreSQL | Users + Bookings (Prisma) |

```
Browser (Netlify)
    │  HTTPS + JWT
    ▼
API (Render)  ──►  PostgreSQL (Render)
```

---

## 1. Local development

### Prerequisites
- Node.js 18+
- Docker Desktop (for local Postgres) **or** any PostgreSQL instance

### Start database

```bash
docker compose up -d
```

Connection string (already in `backend/.env`):

```
postgresql://gharseva:gharseva@localhost:5432/gharseva?schema=public
```

### Start API

```bash
cd backend
npm install
npx prisma db push
npm run dev
```

API: `http://localhost:4000`  
Health: `http://localhost:4000/health`

### Start frontend

Serve the `frontend` folder (do **not** rely on `file://` for production-like testing):

```bash
# Option A — Python
cd frontend
python -m http.server 5500

# Option B — npx
npx --yes serve frontend -l 5500
```

Open `http://localhost:5500`

## 2. Deploy API + Database on Render

### Option A — Blueprint (recommended)

1. Push this repo to GitHub.
2. [Render Dashboard](https://dashboard.render.com) → **New** → **Blueprint**.
3. Connect the repo. Render reads `backend/render.yaml` (or root `render.yaml` if you move it).
4. Create the **PostgreSQL** database and **Web Service**.
5. After the first deploy, set environment variable on the web service:

| Key | Value |
|-----|--------|
| `CORS_ORIGINS` | `https://YOUR-SITE.netlify.app` |
| `JWT_SECRET` | Auto-generated by blueprint, or set your own long secret |
| `DATABASE_URL` | Linked from Postgres (automatic if blueprint) |

6. Build command:

```text
npm install && npx prisma generate && npx prisma db push
```

7. Start command:

```text
npm start
```

8. Root directory: `backend`  
9. Health check path: `/health`

### Option B — Manual

1. **New → PostgreSQL** (Free) → note the **Internal Database URL**.
2. **New → Web Service** → connect repo → root dir `backend`.
3. Environment:

| Key | Value |
|-----|--------|
| `DATABASE_URL` | Internal Database URL from Postgres |
| `JWT_SECRET` | Long random string (32+ chars) |
| `CORS_ORIGINS` | `https://charming-horse-0b7ee8.netlify.app` |
| `NODE_ENV` | `production` |

4. Build / start as above.

Copy your public API URL, e.g. `https://ghar-seva-api.onrender.com`.

> Free web services on Render **spin down** after idle time; first request may take ~30–60s.

## 3. Deploy frontend on Netlify

1. [Netlify](https://app.netlify.com) → **Add new site** → import from Git.
2. Settings:

| Setting | Value |
|---------|--------|
| Base directory | `frontend` |
| Build command | `node inject-config.js` |
| Publish directory | `frontend` (or `.` if base is already `frontend`) |

If using the **root** `netlify.toml`, base is already set to `frontend`.

3. **Site settings → Environment variables**:

| Key | Value |
|-----|--------|
| `API_BASE_URL` | `https://YOUR-RENDER-API.onrender.com` |

4. Deploy.

5. Update Render `CORS_ORIGINS` to include the Netlify URL:

```text
https://your-site.netlify.app
```

6. Redeploy the Render service if needed.

## 4. API reference

### Auth

| Method | Path | Auth | Body |
|--------|------|------|------|
| POST | `/api/auth/signup` | No | `{ name, email, phone, password, location? }` |
| POST | `/api/auth/login` | No | `{ email, password }` |
| GET | `/api/auth/me` | Bearer JWT | — |
| PATCH | `/api/auth/me/location` | Bearer JWT | `{ location }` |

### Bookings

| Method | Path | Auth | Body |
|--------|------|------|------|
| GET | `/api/bookings` | Bearer JWT | — |
| POST | `/api/bookings` | Bearer JWT | See below |
| GET | `/api/bookings/:id` | Bearer JWT | — |

Create booking body:

```json
{
  "serviceId": "plumbing",
  "serviceName": "Plumbing Services",
  "address": "12 MG Road",
  "city": "Hyderabad",
  "pincode": "500001",
  "propertyType": "Apartment / Flat",
  "notes": "Optional",
  "visitDate": "2026-07-20",
  "visitTime": "10:00 AM",
  "paymentMethod": "upi"
}
```

Pass JWT:

```http
Authorization: Bearer <token>
```

Frontend stores the token in `localStorage` (`ghar_seva_token`).

## 5. Security notes

- Passwords are hashed with **bcrypt** (12 rounds).
- Sessions use **JWT** (default 7 days). Set `JWT_EXPIRES_IN` if needed.
- Never commit real `.env` files (see `.gitignore`).
- Payment UI is a **demo checkout** — integrate Razorpay/Stripe for real payments later.
- Rotate `JWT_SECRET` if it is ever leaked (users must log in again).
